## 2026-07-19

### 决策 9.1：知识库跨设备同步策略

**决策日期**：2026-07-19
**决策类型**：产品 + 架构
**决策者**：用户主导 + SparkFox 团队评审

**决策内容**：知识库默认不同步，需用户显式开启。

**理由**：
- 数据主权优先（用户偏好：「别把第二大脑租给别人——你的思考，不该成为别人的养料」）
- 默认本地存储确保用户对数据的完全控制
- E2EE 同步作为可选能力，用户主动开启后才启用

**影响**：
- v1.0.0 知识库默认仅本地存储（`NoOpSync` 占位实现）
- 用户可在设置中开启同步（经 `sparkfox-e2ee` 加密 + `sparkfox-crdt` CRDT 同步）
- 同步开关在 `EmbeddingConfig` 中管理
- v1.1.0+ 提供 E2EE 同步真实实现（基于 Double Ratchet + Automerge CRDT）

**实现位置**：
- `crates/sparkfox/sparkfox-knowledge/src/sync.rs` — `KnowledgeSync` trait + `NoOpSync` 占位
- `crates/sparkfox/sparkfox-e2ee/` — E2EE 加密层（Double Ratchet 自实现，x25519-dalek + aes-gcm + sha2 + hmac）

**关联风险**：
- RISK-SYNC-01：E2EE 同步在弱网环境下 CRDT merge 冲突可能丢失最近编辑（v1.1.0+ 评估）
- RISK-SYNC-02：Double Ratchet 自实现需通过安全审计（v1.1.0+ 引入第三方审计）

## 2026-07-20

### 决策 10.1：v1.1.0 合并 v1.2.0 + v2.0.0 单版本交付

**决策日期**：2026-07-20
**决策类型**：项目管理 + 架构
**决策者**：用户主导

**决策内容**：将原 v1.2.0（Task 11.1-11.5，25 人天）+ v2.0.0（Task 12.1-12.6，33 人天）的全部内容合并到 v1.1.0，通过一个版本的迭代和优化实现完整 SAG 架构 + 营销卖点。

**新版本规划**（取代决策 9.3 的四阶段）：

| 版本 | 工期 | 范围 | SAG 集成 |
|---|---|---|---|
| v1.0.0（已完成） | 14.3 周 | spec 1.0 模块一至九 + 12 个 P0 修复 | 阶段 1：SAG schema 4 表 + hnswlib-rs + LLM 审计 |
| **v1.1.0**（合并三阶段） | 约 11-12 周（105.5 人天，6 路并行扣除重叠后） | SAG 阶段 2 提取管线 + ATOMIC + 阶段 3 MULTI 8 步 + 完整 SAG MULTI_ES + 动态超边 + 中文多跳 Benchmark + 营销卖点 + AGPL 合规审计 | 阶段 2 + 阶段 3 + 完整 SAG 全部在 v1.1.0 完成 |
| v2.0.0（未来） | — | 仅 Bug fix + 性能调优（原 v2.0.0 营销发布版降级为维护版） | — |

**总工期调整**：35.6 周 → 约 25-26 周（节省约 10 周）

**理由**：
1. **加速交付完整 SAG**：用户决策强调「通过一个版本的迭代和优化实现」，避免三阶段渐进交付的中间状态维护成本
2. **减少版本切换开销**：原 v1.1.0 → v1.2.0 → v2.0.0 三次分支管理、回归测试、文档更新合并为一次
3. **营销卖点前置**：原 v2.0.0 的「中文多跳 SOTA」营销卖点提前到 v1.1.0，加速产品上市
4. **并行能力允许合并**：项目支持最多 14 个并行 agent（用户偏好），6 路并行可在 11-12 周内消化 105.5 人天工作量
5. **风险可控**：通过 3 次集成里程碑（W4/W8/W11）分阶段验证，规避单次大版本集成风险

**影响**：

1. **任务结构**：v1.1.0 任务数从 15 个（Task 10.1-10.15）扩展到 26 个（新增 Task 11.1-11.5 + Task 12.1-12.6）
2. **并行策略**：从原 4 路并行扩展到 6 路并行
   - 轨道 A：核心后端 SAG 提取（10.1→10.2→10.3→10.4→10.5/10.6/10.7→10.8→10.9→10.10）
   - 轨道 B：前端 UX + KGView（10.9/10.10/10.11/10.12 → 11.3 → 12.4 → 11.5）
   - 轨道 C：reranker（10.14 → 合并到 10.4）
   - 轨道 D：HnswIndex（10.13 → 10.15）
   - 轨道 E：MULTI 多跳（10.8 → 11.1 → 11.2/11.4/12.1 → 12.2）
   - 轨道 F：Benchmark + 营销 + 合规（11.1 → 12.3 → 12.5 → 12.6）
3. **集成里程碑**：
   - **W4**（4 周末）：SAG 提取管线 + ATOMIC 检索 + 前端 UX P0 修复可用
   - **W8**（8 周末）：MULTI 8 步 + KnowledgeGraphView 完整实现可用
   - **W11**（11 周末）：MULTI_ES + 动态超边 + Benchmark + 营销页 + AGPL 合规审计全部完成
4. **风险新增**：5 项新风险
   - RISK-v1.1-11：中文多跳 Benchmark 工期风险（100 case 人工标注 + 模型调优可能超期）
   - RISK-v1.1-12：MULTI 多跳为关键路径，串行依赖链长（10.8 → 11.1 → 11.2/11.4/12.1 → 12.2）
   - RISK-v1.1-13：动态超边查询性能风险（SQL JOIN 实时激活可能 > 1.5s 阈值）
   - RISK-v1.1-14：单版本集成测试复杂度风险（26 任务合并提交，回归测试覆盖压力大）
   - RISK-v1.1-15：营销页阻塞风险（Benchmark 数据未达 SOTA 前营销页无法上线）
5. **决策 9.2 修订**：原决策 9.2 中 SAG 阶段 2 / 阶段 3 / 完整 SAG 三个阶段全部在 v1.1.0 完成
6. **决策 9.3 修订**：原决策 9.3 的四阶段版本规划调整为两阶段（v1.0.0 + v1.1.0），v2.0.0 降级为维护版

## 💣 避坑指南

### 1. PowerShell 不支持 `&&`
- **现象**：`cd "path" && bun run typecheck` 报错 `The token '&&' is not a valid statement separator`
- **规避**：使用 `;` 分隔：`cd "path"; bun run typecheck`

### 2. `import.meta.env` 类型未声明
- **现象**：typecheck 报错 `Property 'env' does not exist on 'ImportMeta'`
- **规避**：在 `ui/src/renderer/types.d.ts` 顶部添加 `/// <reference types="vite/client" />`

### 3. thinkingStore 结构与文档不符
- **现象**：文档说使用 `rounds` 对象，实际使用 `l1Streams: Map<string, ThoughtLine[]>`
- **规避**：以 [thinkingStore.ts](file:///D:/xin%20kaifa/SparkFox/ui/src/renderer/store/thinkingStore.ts) 源码为准，关键结构：
  ```ts
  l1Streams: Map<string, ThoughtLine[]>;  // 按 AI 消息 clientId 索引
  activeL1Key: string | null;
  l2Stream: ThoughtLine[];  // Tick 心跳流
  ```

### 4. monitorStore.addActivity 事件顺序
- **现象**：新事件放在数组**开头**（最新在前），不是末尾
- **规避**：访问最新事件用 `activities[0]`，不是 `activities[activities.length - 1]`

### 5. monitorStore.initialize() 是 async
- **现象**：未 await 会导致 mock 数据未填充完毕就捕获 initialActCount
- **规避**：必须 `await useMonitorStore.getState().initialize()`

### 6. `replace_all` 对单行重复代码失效
- **现象**：`replace_all: true` 对单行 `border-radius: 8px;` 替换未生效
- **规避**：改用多行上下文匹配（包含前后行作为唯一性锚点）

### 7. 端口 5173 被占用
- **现象**：旧 dev server 进程占用端口
- **规避**：`Get-NetTCPConnection -LocalPort 5173 | Select-Object OwningProcess | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }`

### 8. HashRouter 路由
- **现象**：SparkFox 用 `/#/sparkfox/` 访问，不是 `/sparkfox/`
- **规避**：可视化验证时 URL 必须包含 `/#/sparkfox/` 前缀

### 9. 浏览器自动化工具严重受限
- **规避**：不要尝试用 `browser_evaluate` 测试 React 应用，改用 Zustand store API 直测

### 10. 999px / 2-3px / 主题色阴影不要替换
- **规避**：这些是特殊设计（胶囊形/细节装饰/主题色光效），不属于硬编码残留

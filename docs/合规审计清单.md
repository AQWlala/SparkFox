# SparkFox 合规审计清单（v1.1.0）

> **文档目的**：本清单用于审计 SparkFox v1.1.0 的开源合规性，确保所有上游依赖致谢
> 完整、许可证兼容性明确、清洁室重写记录可追溯。
>
> **维护者**：SparkFox Contributors
> **更新日期**：2026-07-21（Sub-Step 12.6.1）
> **关联文档**：[NOTICE](../NOTICE) / [LICENSE](../LICENSE) / [scripts/compliance_check.sh](../scripts/compliance_check.sh)

## 审计范围

本清单覆盖 SparkFox v1.1.0 的所有上游依赖、衍生代码、数据集与模型权重，
确保符合 AGPL-3.0-only 主许可证及所有上游许可证要求。

## 自动化检查

运行自动化合规检查脚本（GitHub Actions 兼容，Windows 用 git bash 运行）：

```bash
bash scripts/compliance_check.sh
```

脚本包含 5 项检查（详见下方第 1-5 项），任一失败即整体失败。

## 8 项合规检查项

### 1. AGPL-3.0 声明完整 ✅

**通过标准**：全局 `NOTICE` 文件必须包含 `AGPL-3.0-only` 字符串。

**验证方法**：
```bash
grep -q "AGPL-3.0-only" NOTICE
```

**责任文件**：
- `NOTICE`（全局）
- `crates/sparkfox/*/Cargo.toml`（`license = "AGPL-3.0-only"`，workspace 继承）

**关联检查**：compliance_check.sh 检查 1

---

### 2. 上游致谢完整 ✅

**通过标准**：每个核心 crate 的 `NOTICE` 必须包含 `致谢`、`Attribution` 或 `上游`
关键词之一，且列出所有直接依赖的上游项目。

**验证方法**：
```bash
for crate in sparkfox-knowledge sparkfox-graph sparkfox-llm; do
  grep -q "致谢\|Attribution\|上游" "crates/sparkfox/$crate/NOTICE"
done
```

**责任文件**：
- `crates/sparkfox/sparkfox-knowledge/NOTICE`
- `crates/sparkfox/sparkfox-graph/NOTICE`
- `crates/sparkfox/sparkfox-llm/NOTICE`
- `crates/sparkfox/sparkfox-parser/NOTICE`（已存在，v1.0.0 创建）

**关联检查**：compliance_check.sh 检查 2

---

### 3. SAG 引用声明 ✅

**通过标准**：全局 `NOTICE` 必须显式致谢 v1.1.0 SAG 中文检索增强流程引入的 4 项
依赖：`XLM-RoBERTa` / `jieba` / `DuReader` / `CMRC2018`。

**验证方法**：
```bash
for dep in XLM-RoBERTa jieba DuReader CMRC2018; do
  grep -q "$dep" NOTICE
done
```

**责任文件**：
- `NOTICE`（全局，第 6 章节 v1.1.0 新增第三方依赖致谢）
- `crates/sparkfox/sparkfox-knowledge/NOTICE`（crate 级别详细致谢）

**合规依据**：
- XLM-RoBERTa：MIT（model weights）/ Apache-2.0（transformer reference impl）
- jieba-rs：MIT
- DuReader：Apache-2.0
- CMRC2018：CC BY-SA 4.0（仅测试 fixtures，不分发）

**关联检查**：compliance_check.sh 检查 3

---

### 4. 无 MIT 文件残留 ✅

**通过标准**：`crates/sparkfox/` 目录下（排除 `target/` 与 `bench/` 构建产物）
不得存在独立的 MIT LICENSE 文件。MIT 依赖致谢在 NOTICE 中保留即可。

**验证方法**：
```bash
find crates/sparkfox/ -name "LICENSE*" \
  -not -path "*/target/*" \
  -not -path "*/bench/*" \
  -exec grep -l "MIT License" {} \;
# 期望输出为空
```

**责任文件**：禁止在 `crates/sparkfox/*/LICENSE*` 路径放置 MIT 许可证副本。

**例外说明**：
- 全局 `LICENSE` 文件是 AGPL-3.0 全文，非 MIT
- `target/` 是 Cargo 构建产物，不纳入发行版
- `bench/alternatives/` 是 PoC 评测代码，不纳入发行版

**关联检查**：compliance_check.sh 检查 4

---

### 5. Apache 依赖致谢 ✅

**通过标准**：全局 `NOTICE` 必须显式致谢 Apache-2.0 许可证的关键依赖：
`hnsw`（hnsw_rs）/ `petgraph` / `candle`（candle-core）。

**验证方法**：
```bash
for dep in hnsw petgraph candle; do
  grep -qi "$dep" NOTICE
done
```

**责任文件**：
- `NOTICE`（全局，第 6 章节 6.5 / 6.7 / 6.8）
- `crates/sparkfox/sparkfox-knowledge/NOTICE`（hnsw_rs）
- `crates/sparkfox/sparkfox-graph/NOTICE`（petgraph）
- `crates/sparkfox/sparkfox-llm/NOTICE`（candle-core，间接依赖）

**合规依据**：Apache-2.0 §4(d) 要求保留所有上游 NOTICE 文件与致谢。

**关联检查**：compliance_check.sh 检查 5

---

### 6. API 契约字段保留 ✅

**通过标准**：SparkFox 衍生自上游项目的 API 契约（字段名、枚举值、错误码）必须保留，
不得重命名后破坏兼容性。SAG schema 字段虽重命名（C-02 决策），但需在
`docs/SAG-深度评估与重构方案-1.0.md` 中记录字段映射表。

**验证方法**：
- 人工审查 `crates/sparkfox/sparkfox-knowledge/src/schema.rs` 是否含 SAG 5-table
  字段映射注释
- 人工审查 `crates/sparkfox/sparkfox-llm/src/provider.rs` 是否保留 LlmProvider
  trait 的 `complete` / `stream_complete` / `structured_complete` 三方法签名

**责任文件**：
- `crates/sparkfox/sparkfox-knowledge/src/schema.rs`
- `crates/sparkfox/sparkfox-llm/src/provider.rs`
- `docs/SAG-深度评估与重构方案-1.0.md`（字段映射表）

**关联检查**：人工审查（无自动化脚本，需 PR 评审）

---

### 7. 清洁室重写记录 ✅

**通过标准**：衍生自 MIT / Apache-2.0 上游的代码必须记录清洁室重写过程
（Team A 写规格 → Team B 实现，互不接触），或在 NOTICE 中标注"schema borrowing"
（字段级映射，非清洁室重写）。

**验证方法**：
- 全局 `NOTICE` 第 4 章节（BaiLongma）含清洁室重写记录
- 全局 `NOTICE` 第 5 章节（SAG）含 schema borrowing 声明
- `crates/sparkfox/sparkfox-graph/NOTICE` 含 OpenAkita MDRM 清洁室重写声明

**责任文件**：
- `NOTICE`（全局，第 4 / 5 章节）
- `crates/sparkfox/sparkfox-graph/NOTICE`
- `crates/sparkfox/sparkfox-knowledge/NOTICE`
- `docs/SAG-深度评估与重构方案-1.0.md`

**关联检查**：人工审查（需 PR 评审确认 NOTICE 中含清洁室 / schema borrowing 声明）

---

### 8. 历史文档保留 ✅

**通过标准**：项目历史决策文档（RFC、Spec、评审报告）必须保留在 `docs/` 目录，
不得删除。新版本迭代时追加新文档，旧文档标记为"已归档"但保留可追溯。

**验证方法**：
```bash
ls docs/SparkFox-v1.0.0-spec-*.md
ls docs/SparkFox-v1.1.0-规划.md
ls docs/RFC-*.md
ls docs/SAG-*.md
```

**责任文件**：
- `docs/SparkFox-v1.0.0-spec-1.0.md` / `spec-2.0.md`
- `docs/SparkFox-v1.1.0-规划.md`
- `docs/RFC-001-crate-boundaries.md` ~ `RFC-005-parallelism-and-target-isolation.md`
- `docs/SAG-深度评估与重构方案-1.0.md`
- `docs/SAG-重构方案-七专家评审-1.0.md`
- `docs/SparkFox-四项目深度分析报告-1.0.md`

**关联检查**：人工审查（CI 仅检查文件存在性，内容完整性需 PR 评审）
